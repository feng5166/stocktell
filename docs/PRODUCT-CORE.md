# StockTell 产品内核(定稿 · 权威版)

> **这是 StockTell 的产品北极星。技术 / 产品 / 美术,后续讨论问题、开发任何新功能,先对照本文检查是否符合内核。**
> 最后更新:2026-07-02。精简版见根目录 `CLAUDE.md` 与 `README.md`;本文为完整定稿。

---

## 一、核心结论

StockTell 不是"股票信息产品",而是:

> **全球事件到本土资产的因果链推理工具。**

"AI 时代的 Bloomberg"是长期愿景;落到执行,核心是:

> **把一个外部事件,逐跳推演成产业链影响、资产映射和可验证证据。**

长期方向:不是另一个行情软件,也不是新闻聚合,而是一个 **AI-native 的产业理解工具**。

---

## 二、愿景与第一阶段定位

- **长期愿景**:StockTell = AI 时代的 Bloomberg(数据终端/行情/新闻/交易员工作台/专业金融基础设施的心智)。但这是愿景,不是第一阶段的产品边界。
- **第一阶段定位**:AI-native 的产业理解工具。切口锐利到一句话:
  > **全球事件 → 产业链传导 → 本土资产映射。**
  > 从一件全球大事,推演出它会影响中国哪些产业链和股票,并给出每一跳依据。

---

## 三、产品公式(= 验收标准)

```text
StockTell = 事件输入 × 产业链图谱 × 多跳推理 × 资产映射 × 证据验证
```

口语版:**从一件全球大事,推演出它会影响中国哪些产业链和股票,并给出每一跳依据。**

任何新功能都先问:
> **它有没有帮助用户完成"事件 → 链条 → 标的 → 依据"的推理?**
> 没有就谨慎做,甚至砍掉。

---

## 四、核心动作:多跳推理(不是单跳、不是新闻聚合)

- **反例(太浅、无壁垒)**:`英伟达上涨 → A股 AI 芯片上涨`。
- **正例(核心壁垒)**:
```text
OpenAI 发布 GPT-6
  ↓ 模型能力提升 / 推理成本下降
  ↓ AI 应用调用量上升
  ↓ 算力与数据中心需求上升
  ↓ GPU / ASIC 需求上升
  ↓ HBM / 先进封装 / 液冷 / 光模块 / 电力设备升温
  ↓ 国内映射:海光信息、长电科技、中际旭创、英维克、沃尔核材 …
```

StockTell 要回答的 5 个问题:
1. 这件事为什么重要?
2. 它会沿哪些产业链环节传导?
3. 国内哪些股票可能受影响?
4. 每一跳推理的依据是什么?
5. 历史上类似链路是否有效?

---

## 五、核心交付三要素(每个事件/产业链/个股场景尽量都给)

```text
一句话判断 · 产业链热力 · 可核实因果链
```

### 1. 一句话判断(结论先行)
回答:利好哪条链?影响哪个环节?今天该盯什么?强影响/弱影响/情绪映射?
- ✅ 示例:"OpenAI 发布 GPT-6,短期更可能利好 AI 推理基础设施链,重点关注算力、先进封装、液冷、光模块和数据中心电力环节。"
- ❌ 禁止交易指令:"建议买入光模块 / 明天中际旭创可能上涨 / 低吸海光"。
- ✅ 允许:"光模块环节关注度升温 / 国产算力链条可能受情绪映射 / 需观察美股算力链与 A股联动强度"。

### 2. 产业链热力
让用户一眼看到:哪些环节升温 / 降温 / 只是情绪映射 / 有基本面证据支撑。早期可先用**结构化矩阵**(不必复杂可视化),关键是信息结构正确。

### 3. 可核实因果链(可信度核心)
每一跳尽量给:推理逻辑、证据来源、更新时间、置信度、是否 AI 推理、是否为假设。
- 不必每跳都是强证据,**允许中低置信度,但必须标出来**。明确标注不确定性反而更可信。

---

## 六、StockTell 不是什么

1. **不是新闻聚合器**:不做"英伟达上涨 / OpenAI 发布 / 苹果加码 AI"式信息堆叠——东财/同花顺/雪球/财联社更快更全。价值在"为什么重要/传导路径/国内标的/依据/历史是否有效"。
2. **不是行情软件**:不抢盘口/分时/K线/龙虎榜/F10/三表/技术指标/资金流排行。这些只作推理证据,不做主界面主心智。
3. **不是荐股工具**:不输出买入/卖出/低吸/冲/目标价/明天涨/必涨/短线机会。只做产业链解释、影响映射、历史复盘、证据来源、置信度、风险提示。

---

## 七、三条产品铁律(硬边界)

- **铁律 1:绝不做新闻聚合。** 只收事件,不堆新闻;必须从"发生了什么"推演到"为什么重要/影响哪条链/传导到哪个环节/国内映射标的/依据/历史是否有效"。
- **铁律 2:绝不输出无依据结论。** 所有判断能回溯到:事件/数据/公告/财报/产业链关系/历史复盘/官方来源/价格变化。无强证据处必须标"推理假设 / 低置信度 / 待验证"。不伪装确定性。
- **铁律 3:绝不做交易指令。** 不输出买入/卖出/目标价/明日涨跌。只做产业链解释、影响映射、历史复盘、风险提示、观察点、置信度。**从内容、分享卡、页面文案、AI 对话一起执行。**

## 八、标准内容单元(每条内容按固定模板生成)

```text
事件:OpenAI 发布 GPT-6
一句话判断:利好 AI 推理基础设施链,重点关注算力、先进封装、液冷、光模块、数据中心电力。
产业链热力:模型层↑ 推理需求↑↑ GPU/ASIC↑ HBM↑ 先进封装↑ 液冷↑ 光模块↑ AI应用↑
因果链:
 1. GPT-6 提升模型能力 → 扩大 AI 应用场景
 2. AI 应用调用量上升 → 推理算力需求增加
 3. 推理算力上升 → 数据中心 GPU/ASIC 需求
 4. 高密度算力 → HBM/先进封装/液冷/光模块/电力更高要求
 5. 国内映射到相关 A股产业链标的
国内映射:海光信息(国产算力芯片)/ 长电科技(先进封装)/ 中际旭创(光模块)/ 英维克(液冷)/ 沃尔核材(高速互连)
可信度:每跳 references + 历史同类复盘 + 置信度 + 数据更新时间 + 免责声明
```

## 九、核心数据对象:ReasoningChain(多跳推理必须结构化,不能只是 AI 长文)

```ts
type ReasoningChain = {
  eventId: string
  eventTitle: string
  eventType: 'model_release' | 'earnings' | 'policy' | 'product_launch' | 'capex' | 'supply_chain' | 'macro'
  summary: string
  conclusion: {
    oneLiner: string
    affectedChains: string[]
    watchPoints: string[]
    confidence: 'high' | 'medium' | 'low'
  }
  hops: Array<{ order: number; from: string; to: string; logic: string; evidence: Reference[]; confidence: 'high' | 'medium' | 'low' }>
  heatmap: Array<{ chainId: string; segment: string; direction: 'up' | 'down' | 'mixed' | 'neutral'; intensity: 1|2|3|4|5; reason: string }>
  stockMappings: Array<{ code: string; name: string; market: 'A'|'HK'|'US'; chainId: string; segment: string; relation: 'direct'|'indirect'|'sentiment'|'weak'; reason: string; confidence: 'high'|'medium'|'low' }>
  references: Reference[]
  generatedAt: string
  updatedAt: string
}
```

**为什么必须结构化**:一旦有 `ReasoningChain`,首页/事件详情/产业链页/个股页/分享海报/AI 对话上下文/历史复盘/References/自选关联全部复用。**不要把多跳推理只写在 Markdown 里,要变成核心数据结构。**

## 十、页面体系

- **首页**:从"今日简报"升级为**今日最重要的 3 条全球事件推理链**。每条卡片固定给:一句话判断 / 产业链热力 / 3 个核心受影响环节 / 3-5 个国内映射标的 / 查看完整因果链。
- **推理链详情页 `/event/[id]`(或 `/insight/[id]`)**(新内核主展示页):事件 / 一句话判断 / 产业链热力 / 多跳因果链 / 国内映射标的 / 每跳 references / 历史同类复盘 / 继续追问 / 免责声明。
- **产业链页 `/chain/[id]`**:名称 / 今日热力 / 最近触发事件 / 相关推理链 / 成分股 / 相关美股 / 国内映射 / 历史联动有效率 / 订阅。承担:分享落地页 + SEO + 转化。
- **个股详情页**:一只股票的**产业链档案 + 近期事件解释 + 历史联动记录**——基础信息 / 所属链 / 产业链位置 / 关联美股 / 近期触发事件 / 相关推理链 / 影响强度 / 历史联动 / references / AI 追问。

---

## 十一、分享 / 对话 / References

- **分享升级**:推理链是分享内容,产业链是承接页面。事件推理卡(负责传播)+ 产业链落地页(负责沉淀)+ 个股卡(负责好奇心补充)。事件卡:全球事件→三跳传导→受影响链→国内映射→扫码看完整因果链。
- **对话**:围绕推理链**追问**(不是空白通用聊天框)。可问:这条链最弱一跳?为什么 GPT-6 影响液冷?哪些是直接映射/哪些只是情绪映射?历史准不准?我自选里哪些和这条链相关?上下文来自:当前事件/推理链/产业链/个股 + 用户自选 + references + 历史复盘。
- **References = 生命线**:每跳给 证据类型/来源/更新时间/置信度/是否AI推理/是否假设。证据类型:官方公告/财报/纪要/监管文件/行情/公司新闻/产业链数据库/历史复盘/第三方研究。展示标注:强/中/弱证据、推理假设、待验证。

---

## 十二、优先级

- **P0 推理链详情页**:先把"事件→链条→标的→依据"主链路产品化。
- **P0 产业链热力图**:先结构化矩阵起步,让用户一眼看出升温/降温/分化。
- **P0 References 数据结构**:必须马上做,否则"可核实因果链"只是口号。
- **P1**:链页 `/chain/[id]` 重构、对话(围绕推理链)、分享卡升级(事件推理卡+产业链落地页+个股卡)。

---

## 十三、最终判断(关键提醒)

> **多跳推理必须产品化、结构化、可验证,不能只变成 AI 长文。**

只让模型写一段"GPT-6 利好算力/HBM/液冷/光模块……"别人很快能抄。真正难抄的是这套系统:
**事件库 · 产业链图谱 · 推理链对象(ReasoningChain)· 每跳 references · 置信度标注 · 国内资产映射 · 历史复盘验证 · 用户自选股关联。**

这套做起来,StockTell 才从"AI 资讯页"变成"产业理解工具"。

## 检查清单(讨论问题 / 开发新功能前逐条过)

- [ ] 它服务"事件 → 产业链 → 个股 → 依据"这条推理链吗?(不服务就砍)
- [ ] 有没有沦为新闻聚合 / 行情看板 / 资讯堆叠?(是→违铁律1)
- [ ] 结论是"一句话判断 + 对我自选的票的影响",而非丢一堆数据?
- [ ] 每一跳/每个结论可回溯到依据?无强证据处标了置信度/假设?(否→违铁律2)
- [ ] 有没有输出买卖/目标价/涨跌预测?(有→违铁律3)
- [ ] 多跳推理是结构化的 `ReasoningChain`,还是又写成一段 AI 长文?
