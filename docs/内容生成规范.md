# 内容生成规范(简报 / 散户怎么想 / 早报)

> 面向所有改简报生成、推送文案的人。实现见 `src/lib/generate.ts`(简报+retailTake)、`src/lib/morning-brief.ts`(早报)、`src/lib/digest.ts`(邮件)。

## 合规红线(违反即废)
- 禁止任何操作指令性措辞:买入/卖出/建议买/推荐/抄底/满仓/加仓/清仓 等一律不准。
- 可点透机会与陷阱、提示风险,但**绝不下买卖结论**。
- 正文不写"不构成投资建议/历史规律不代表未来"之类免责(页面底部已有,别重复)。

## 「散户怎么想」(retailTake)—— 零容忍:不出现具体涨跌数字
**铁律:retailTake 里绝不出现任何具体涨跌数字(A股、美股都不行)**。
不要写"涨了3个多点""跌6个点""涨超5%""微涨0.87%""跌不到2%"这类,一律换成定性词:
大涨 / 重挫 / 逆势走强 / 没怎么跟跌 / 跌得更狠 / 基本同步 / 相对抗跌。

**为什么**:retailTake 在**盘前生成**(A股还没开盘),拿到的 A股涨跌是**上一交易日**的;而页面顶部行情是 `<LiveQuote>` **实时**拉的。两个不同时点的数字摆一起必然对不上(实测踩坑:文案写"中微涨了3个点",实时行情却 +10.58%)。只做定性,就不会和实时行情打架。

**三道保险(`src/lib/generate.ts`)**:
1. **Prompt**:`SYSTEM_PROMPT` 硬性禁止任何涨跌数字,只定性。
2. **检测**:`hasSpecificMove(text)` 扫 `X% / X个点 / 涨了X / 跌了X`。
3. **回退**:LLM 哪条仍带数字 → 自动换成模板 `buildTake(m)`(模板本身已去掉美股数字,确定性零数字)。

> 改 prompt / 模板时,务必让 `buildTake` 与 LLM 输出都过 `hasSpecificMove` 这关。新增的文案位(早报/微信/邮件)若也会和实时行情同屏,沿用同一条零数字铁律。

## 数据时点一致性(通则)
盘前生成的叙述 vs 页面实时行情存在天然时间 gap —— 凡是"叙述里写死的数值"都可能与实时数据打架。原则:**能定性就别写死数值;必须给数值时,标清"截至 X / 盘前 / 上一交易日"**。各模块数据更新周期见对话记录的「数据更新周期报告」。
