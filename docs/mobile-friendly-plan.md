# 移动端友好改造方案

> 目的:让 StockTell 在手机上真正好用——散户主力在手机看盘,移动端体验是命脉。
> 现状:基础(viewport / PWA / 页面卡片化)已扎实,短板**全集中在"表格类"展示**:小屏只能横向滚动。
> 范围:面向用户的页面与组件。断点统一以 Tailwind `sm`(640px)作手机/桌面分界。

---

## 一、现状评估

### 已做对(保持,别推倒)
- **viewport**:`device-width` + `viewportFit: cover`(适配刘海/灵动岛);**PWA**:`manifest` + `appleWebApp`(可加桌面)。
- **首页 / 个股详情 / 早报 / 简报 feed**:卡片式 + `max-w-3xl` 全宽,手机已友好。
- 统计卡 `grid-cols-2 sm:grid-cols-4`、统一 `SiteHeader`、`BriefingFeed` 无限滚动 + 早报链接。

### 短板(待改)——清一色是"表格"
| 位置 | 文件 | 问题 | 严重度 |
|---|---|---|---|
| 股票池列表 `StockTable`(11 列) | `components/Dashboard.tsx` | `overflow-x-auto` 横滚,涨跌/定位/状态要左右划 | 🔴 P0 |
| 战绩 `OutcomeTable`(6 列) | `app/track/page.tsx` | 同上横滚,手机查命中明细别扭 | 🔴 P0 |
| 特征矩阵 `FeatureMatrix`(5 列) | `components/Dashboard.tsx` | **表格没包 `overflow-x-auto`**,小屏可能撑破整页布局 | 🟡 P1 |
| tab(4 个)+ 筛选区 | `components/Dashboard.tsx` | 窄屏 tab 偏挤、筛选 chips 占掉小半屏 | 🟡 P1 |
| 触摸目标(☆ / 小 chip) | 多处 | 点击区 < 44px,手指难点准 | 🟢 P2 |
| 关联图谱 / 预期差 | `components/Dashboard.tsx` | `grid grid-cols-1 md:grid-cols-2`,手机单列,基本 OK | ✅ |

**判断:移动端友好的关键一仗 = 把"表格类"在小屏改成卡片式。** 其余早已卡片化、手机友好。表格是唯一也是最高 ROI 的硬伤。

## 二、详细改造方案

### 1. 股票池 `StockTable` → 移动端卡片(🔴 P0,最高 ROI)
**现状**:11 列 `<table>` 包在 `overflow-x-auto`,手机横滚才能看全。
**改法**:同一份数据,**两套渲染按断点切换**——桌面 `hidden sm:block` 保留表格;手机 `sm:hidden` 每只股票一张卡:
```
┌──────────────────────────────┐
│ ☆  中际旭创  300308    +5.32% │ ← 名+代码在左,涨跌大字在右(红涨绿跌)
│ A股 · 中游 · 光模块/CPO        │ ← 市场/位置/板块 一行小字
│ 全球光模块龙头…                │ ← 核心定位;整卡可点进 /stock/[code]
└──────────────────────────────┘
```
- 关键信息(涨跌)一眼可见,不用横滑;整卡点进详情。
- ☆ 收藏放卡左侧、点击区放大;"散户怎么想"由表格行内展开改为卡片点开/进详情看。
**实现**:把行数据复用,`ReactFragmentRow`(表格行)旁新增 `StockCard`;列表容器手机渲染卡片列表、桌面渲染表格。
**工作量**:0.5–1 天。

### 2. 战绩 `OutcomeTable` → 移动端卡片(🔴 P0)
**现状**:6 列表格(日期/简报/受益股/期待/实际/结果)横滚。
**改法**:手机一条命中记录一张卡,桌面保留表格:
```
🔴 美光盘后大涨15.7%            06-26
兆易创新   期待:涨 · 实际:-1.2% · 没跟上
```
**工作量**:0.5 天。

### 3. 特征矩阵 `FeatureMatrix` → 止血 + 精简(🟡 P1)
**现状**:`<table>` **没有** `overflow-x-auto` 包裹,小屏可能撑破整页。
**改法**:① **立刻止血**——给表格套一层 `overflow-x-auto`(5 分钟,防撑破);② 手机精简为每板块一行紧凑卡(板块名 + A股均涨跌 + 强弱条),桌面保留表格。
**工作量**:0.3 天(止血几分钟)。

### 4. tab 导航 + 筛选区 → 手机紧凑(🟡 P1)
- **tab**(股票列表/关联图谱/特征矩阵/主动发现):窄屏挤。改 `overflow-x-auto whitespace-nowrap` 让它横向可滚,或缩短文案。
- **筛选区**:手机默认**折叠**为一个「筛选 ▾」按钮,展开才显示市场/位置/关系/板块;常用的搜索框留在外面。省掉小半屏首屏空间。
**工作量**:0.5 天。

### 5. 触摸目标 + 细节(🟢 P2)
- 表格/卡片里的 ☆、小 chip:点击区 padding 放大到 ≥ 40px。
- 产业链切换器 chips:手机换行已 OK,确认行距够手指点。
**工作量**:0.3 天。

---

## 三、通用规范
- **断点**:统一 Tailwind `sm`(640px)做手机/桌面分界;两套布局用 `hidden sm:block`(桌面)/ `sm:hidden`(手机)。
- **页面级永不横滚**:表格要么卡片化、要么显式局部 `overflow-x-auto`,绝不让整页出现横向滚动条。
- **触摸目标**:可点元素点击区 ≥ 40px。
- **自测**:Chrome DevTools 切 iPhone SE(375px)、常见安卓(393px),逐页过;有真机更好。

---

## 四、建议执行顺序(低风险 → 高收益优先)
1. **P0**:股票池表格卡片化 → 战绩表卡片化。这俩是散户手机高频场景,做完移动端体验质变。
2. **P1**:特征矩阵止血(套 `overflow-x-auto`)→ tab / 筛选区紧凑。
3. **P2**:触摸目标打磨。

每步 `npx tsc --noEmit` + `npm run build` + DevTools 移动模拟自测;桌面布局保持不变(只在 `sm` 以下叠加移动样式,不动现有桌面表格)。
