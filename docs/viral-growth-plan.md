# StockTell 裂变增长方案 v1（拍板执行版）

> 给老板拍板用。结论先行,取舍明确,合规焊死。两份对抗评审里"该砍/该重构/合规硬改"的结论已真正吸收(见每节末尾的「评审吸收」标注),不再把被否决的设计当卖点。
> 产出方式:多智能体工作流(代码盘点 + 增长基线 → 增长循环/PM落地/合规风控/散户心理 4 视角 → 2 路对抗评审 → 综合)。

---

## 1. TL;DR / 核心主张

**一句话差异**:传统炒股 App 的裂变社交货币是「我赚了」,而 StockTell 既无券商资质、又不追踪个人盈亏、又被合规禁止喊收益——这条路从机制、数据、合规三重被堵死。我们的社交货币只能是「**看懂了 / 敢查账 / 帮到你 / 不焦虑**」。

**核心主张(已对两份评审踩刹车后的版本)**:

- **这不是一套"五环 K 因子裂变体系",而是一个"一张卡的可证伪实验"起步。** 战略方向(社交货币迁移、绿区自愿转发、固定免责)是对的;但节奏、优先级、对用户分享意愿的乐观度全部需要踩刹车。
- **主打循环就 1.5 条**:① **「AI链今日情绪」内容卡自愿转发**(唯一能跑量、零账号依赖、每日有新内容、合规最干净)为主引擎;② **个股利他点对点转发**(CVR 最高但低频)为质量补刀。其余卡和整套邀请/权益体系一律后置。
- **定位口径从「K 因子指数级裂变」降级为「合规绿区内容分发 + 高质量点对点口碑」。** 早期用户基数下 K 无统计意义;喊 K 因子会逆向逼团队为凑数加诱导、加攀比、加利益,正好撞微信红线和反焦虑品牌。

## 2. 前提与判断:现在适不适合大推裂变

**判断:不适合"大推"。现在做的是"小成本可证伪实验",不是放量。** 采取「**先打磨留存 + 小成本验证一张卡,再放量**」,而非"边裂变边迭代"。

三个硬前提(两份评审共同的致命提醒,此前方案默认已解决、实则未解决):

1. **PMF/留存未验证,不能重仓裂变基建。** 第一批几十个用户、日活不稳定时,把 OG 海报 + 短链 + 邀请表 + 权益系统全压上,是典型"增长功能先于 PMF"。**先用真实漏斗数据回答唯一重要的问题:到底有没有人愿意发?发出去有没有人扫?扫进来转不转化?**
2. **主体/资质闸门未过,不能主动放量。** 境内运营主体 + ICP 备案 + 证券律师过审(战绩机制/主体声明)是放量前的硬闸门,尚未落定。裂变本质是把样本量做大;非法荐股是"概率低、后果重、靠同行举报与亏损用户投诉触发"的尾部风险——闸门过前大规模拉新 = 先把"被举报样本量"做大。
3. **域名是唯一存亡资产。** 主域 `stocktell.me` 被微信封 = 产品入口归零。承接域名必须与主站物理隔离(见第 8 节)。

**触发条件(满足才进入下一阶段放量):**

| 阶段 | 进入条件 |
|---|---|
| 阶段 A:一张卡实验(现在) | 无前置,立刻可做(纯绿区内容卡 + og:image + 短链 + 3 个埋点) |
| 阶段 B:补刀 + 留存巩固 | 阶段 A 跑出真实漏斗:有人发卡、有扫码、扫码注册转化率不为零;且站内有稳定日活底座(邮件早报打开率 + PWA 回访达观测阈值) |
| 阶段 C:邀请/权益体系放量 | **主体 + ICP + 律师过审三项全部通过** + 阶段 B 漏斗数据正向 + 月活到达可让 K 因子有统计意义的量级 |

> **评审吸收**:评审1「在 PMF 未验证前重仓裂变基建是功能先于需求」→ 砍成阶段 A 一张卡实验。评审2「主体/ICP/律师是硬闸门,放量动作必须卡在闸门后」→ 邀请/权益体系整体后置到阶段 C。

## 3. 裂变循环设计

每条标注:触发点 → 分享物 → 渠道 → 落地承接 → 回环 → K 杠杆,以及**强弱与优先级**。对用户分享意愿做了诚实降级(吸收两份评审"5 张卡里真能跑的只有情绪卡一张"的判断)。

### 环① 情绪卡内容环 ★主引擎(强 / P0)
- **触发点**:首页「AI链今日情绪」仪表盘旁固定「生成今日情绪卡」按钮,每个交易日早盘用户打开即见。
- **分享物**:全量同款竖版海报——A股 AI 链涨跌家数/均涨跌% + 主力净流入 + 隔夜美股 + 纳指/标普/费半 + 一句"今日温度"。无个人信息。
- **渠道**:朋友圈、股票群。
- **落地承接**:扫码→独立承接域的"今日情绪只读页"→软提示"加自选,明天早报里就有你的票"。
- **回环**:新用户加自选→次日自己收到情绪页→同样看到生成按钮→成为新发卡人。
- **K 杠杆**:优化 i(发卡率)+ R(触达)。海报必须美(认知体面感)、默认带码一键生成(降摩擦)。
- **强弱诚实标注**:这是唯一真能跑量的一张。但"每日复发"是一厢情愿——普通人不会天天发同一类卡刷屏,会被嫌烦;真正爱发的是少数想立"我在看盘"人设的次级 KOC,是小众而非大盘。**定位为"品牌曝光 + 口碑",不指望指数级。**

### 环② 个股利他环 ★质量补刀(中强 / P1)
- **触发点**:个股页 + 深读结果后,「发给关注这只票的朋友」按钮。
- **分享物**:该股一句话解读 + 产业链位置缩略卡,中性第二人称。
- **渠道**:私聊为主。
- **落地承接**:直达该股公开页,首屏"这只票今天该看什么",CTA"加进自选自动帮你盯"。
- **回环**:被邀者加自选→以后也收到该票解读→复用同按钮二次转发。
- **K 杠杆**:CVR 全场最高("朋友专门发给我的、又正好我关心")。
- **强弱诚实标注**:CTR×CVR 最高但**触发极低频**——需同时满足(a)用户在研究这票(b)知道某朋友也拿着(c)当天有动静三者交集。**高质量低频,撑不起主引擎,是补刀不是主力。**文案必须去持仓预设。

### 环③ 邀请激励环(中 / 后置到阶段 C,且极象征化)
- 仅在主体+ICP+律师闸门过后上;且砍到**单级、低封顶、无排行**(见第 5 节)。
- K 杠杆:温和抬 CVR,是注册放大器,不创造内容。没有①②的内容流就没有放大对象。

### 环④ 自选清单克隆环(弱 / 不进路线图,PMF+合规复核双过后才重评)
- 三重死锁(整理成本高 + 被当荐股的心理负担 + 早期无像样自选)+ 第二大合规隐患("可一键克隆 + 规模化传播"客观接近"面向不特定对象提供证券投资建议/带单结构")。**不是 P2,是搁置项。**

> **评审吸收**:
> - 「复盘卡对外背书是最大合规自杀点」→ **复盘卡从裂变素材里彻底删除**,只做站内对已注册用户的信任组件。
> - 「5 张卡只有情绪卡能跑」→ 早报卡/深读卡降级为"站内留存/偶发转发",不作为裂变主力推。
> - 「措辞预设对方持仓」→ 环②文案改"关注 XX / 在看 XX 的朋友",**禁用"你手里的/拿着/我上了"等预设持仓的措辞**。
> - 「清单克隆三重死锁 + 带单风险」→ 环④移出路线图。

## 4. 分享物料规范

**只规范实际要做的两类卡(情绪卡、个股利他卡)。** 复盘卡、早报卡、深读卡、清单卡不进对外裂变素材库。

**通用硬规则(所有卡):**
- **品牌区**:左上 logo + slogan「我不懂产业链,你告诉我怎么想」。
- **免责水印**:**写死为生成模板的固定层,非参数、无删除入口、不可被装饰遮挡、字号可读。**
- **二维码/短链**:右下角二维码,指向**独立承接子域**(带归因参数),不回链到邀请者个人页。
- **文案管控**:**不提供用户自填叠加文本入口**;"复制文案"只给可复制文本、不给可编辑框,末尾带免责;工程不得拼接用户自填买卖性文字。
- **隐私默认**:`showIdentity=false`、`showHoldingHint=false` 为模板不可编辑层。
- **零数字闸**:卡面文案复用现有 `src/lib/generate.ts` 的 `hasSpecificMove` 零数字检测 + 买卖黑名单(prompt/检测/回退三道保险),同一闸过。

### 卡1 ——「AI链今日情绪」卡(P0,发卡频率最高)
- **内容要素**:日期 + A股涨跌家数/均涨跌% + 主力净流入 + 隔夜美股 + 三大指数 + 一句"今日温度"。无个性化、无个人信息。
- **视觉**:温度色块(暖红冷蓝)+ 指数小卡排列,信息密度高不杂。
- **合规文案示例(标题)**:「AI链今日情绪 · 市场一瞥」
- **固定免责水印**:「信息参考,不构成投资建议。市场有风险。StockTell · stocktell.me」
- **do**:用"温度/市场一瞥/信息参考";**don't**:不得写"今天该买/会涨/行情预测"。

### 卡2 ——「个股产业链位置」利他卡(P1)
- **内容要素**:个股名+代码 + 上游/你在这/下游位置图 + 一句"它是干什么的" + 所属概念/ETF。无持仓暗示。
- **视觉**:横向产业链链路图,当前票高亮"你在这"。
- **合规文案示例(标题)**:「它在产业链的位置 · 研究框架」;转发语「关注 XX 的朋友可以看看它在这条链站哪」(中性,非"你手里的")。
- **固定免责水印**:「产业链关系为研究框架梳理,非确认供货关系,不构成投资建议。StockTell · stocktell.me」
- **do**:用"研究框架/位置";**don't**:不提涨跌、不写"推荐这只/你手里的/我上了"。
- **个性化分享前二次确认**:若卡涉及用户自选标的,弹"此卡含你关注的标的,确认分享?"

> **评审吸收**:复制文案只读不可编辑 + 末尾带免责 + 不提供叠加入口;showIdentity/showHoldingHint 为不可编辑工程硬约束 + 禁第一人称持仓措辞。复盘卡/早报卡/深读卡不做对外卡。

## 5. 邀请/激励体系(后置到阶段 C,且极象征化)

**整套邀请/权益体系卡在「主体 + ICP + 律师过审」闸门后,放量前不上。**

- **激励物(全为产品内权益,绝不碰钱/券/股)**:
  - 被邀请者:注册即送 **7 天深读体验**(即时反馈,抬注册转化)。
  - 邀请者:每生效一人 **+小额自选额度(低封顶)**。
  - **砍掉:邀请榜公开攀比、阶梯加码、"邀5人永久解锁深读"、"随机解锁个股深读券"。**
- **邀请码与关系**:每用户一个稳定 `invite_code`(用 code 不用裸 userId 防枚举);落地写 cookie `st_ref` 30 天;注册成功且有 ref → 建 `Invite(inviterId,inviteeId,code,src,status)`。防自邀。
- **防刷与生效判定**:生效 = **注册 + 次日有效登录 + 至少加 1 只自选**;同设备/IP 限流、邮箱验证、24h 无行为不计生效;单邀请者封顶;后台可冻结。(此判定同时是合规净化器,但会拉长链条、砍掉部分真人转化——用增量真实性换数量,刻意取舍。)
- **微信合规结构**:权益**只认"被邀请人注册成功(并满足生效判定)"**,与"转发动作/转发到哪/转发几次"完全脱钩;内容卡转发永远自愿、无任何功能因未分享而锁定。

> **评审吸收**:阶梯/排行/永久解锁 = 利益诱导拉人头的功能等价物(即便奖品是产品权益)→ 砍到单级低封顶无排行;惊喜激励不绑个股解读;整体后置到闸门后。

## 7. 指标体系与目标

**早期不以 K 因子为北极星(小分母下无统计意义,会被噪声误导)。** 改用绝对值薄漏斗 + 留存交叉。

- **阶段 A 北极星(绝对值薄漏斗)**:发卡人数 → 扫码数 → 扫码注册转化率(配合 Umami 现有埋点)。
  - 早期可达基线(观测口径,非承诺):发卡人数 ≥ 活跃用户个位数%(>0 即证明假设成立);扫码 CTR 8–15%;扫码→加自选/注册 CVR 10–20%(以"加自选"为真激活口径)。
- **健康度/防羊毛交叉(最关键)**:**裂变来源用户次周留存 vs 自然用户留存**——显著偏低即标记羊毛,暂停其权益。
- **卡类型 ROI**:按 `card_type` 看带量,集中资源。
- **K 因子使用时机**:月活到达分母有统计意义的量级后,才升级北极星为 **K = 邀请生效注册数 ÷ 活跃发卡用户数**(用 `invite_qualified` 过滤羊毛)。

## 8. 风险与规避(逐条,吸收两份评审硬改)

1. **主体/资质尾部风险**:放量前必过境内主体 + ICP + 律师审。规避:邀请/阶梯/排行等主动放量动作**卡在闸门后**;闸门前只放绿区内容卡自愿转发。
2. **域名存亡风险**:任一链路被判诱导→封主域→微信内所有 stocktell.me 链接全死。规避:**分享/承接域名与主站域名物理隔离**(独立子域或独立域名);微信内承接强制"保存图片+手动发"降级,不做微信 JS-SDK 自动分享闭环;主推 PWA+邮件。
3. **微信诱导分享封禁**:无"分享/集赞/转发才能用"任何硬门槛;激励只认"注册生效",与转发动作脱钩;**砍邀请榜公开攀比、阶梯加码、永久解锁**。
4. **复盘卡合规自杀**:把"联动有效率"做成可批量转发的公开背书卡 = 用历史表现招揽。规避:**复盘卡彻底退回站内、仅对已注册用户展示**,绝不做对外裂变素材;永不换算成百分比/胜率/收益。
5. **清单克隆=带单结构**:可一键克隆 + 规模化传播接近"面向不特定对象提供证券投资建议"。规避:移出路线图,PMF+合规复核双过后才重评。
6. **个性化卡暴露持仓**:`showIdentity/showHoldingHint` 默认 false 为不可编辑模板层;**禁用第一人称持仓措辞**;个性化卡分享前二次确认;个性化卡不作对外裂变主力,只情绪卡这类全量同款对外。
7. **埋点串持仓轨迹**:邀请/归因链路不携带可反查到个人的持仓明细;分享卡 ticker 仅作选题 ROI 聚合,**不与 userId/邀请关系 joinable**。
8. **用户自填文案连带品牌**:**不提供可编辑框/叠加入口**,只给只读文案+免责。
9. **激励成本/刷量**:激励只给产品权益、低封顶;生效判定=注册+次日登录+加自选;来源留存交叉防羊毛。
10. **反焦虑被做成新焦虑**:成就/连续天数身份卡砍掉;若做只保留"累计看懂 N 条产业链科普"纯学习量、无时间连续压力、不公开排名。

## 9. 冷启动运营动作(产品之外的配合)

裂变早期靠运营推一把,不靠机制自转。

- **种子发卡人 = 次级 KOC,不是大盘**:主动识别站内"想立看盘人设"的活跃用户,1 对 1 邀请成为首批发卡人,给非物质身份认可(内测身份、产品建议直通),而非物质奖励。
- **内容节奏**:情绪卡日更级供给(交易日早盘),但**不鼓励同一用户天天刷屏**;运营侧每周挑 1-2 个"今日温度有故事"的交易日重点推。
- **群与渠道**:暂不重建微信群(24h 窗口 + 诱导红线);先用邮件早报 + PWA 推送做留存底座。B站/微博只做产品科普/反焦虑人设内容,**不搬复盘背书卡**。
- **KOC 清单先别碰**:清单克隆环搁置,运营也不主动造"KOC 清单"种子。
