> 读者：营销发行负责人｜决策口径：盯盘搭子定位 + 保守合规｜依据：五专家建议 + 三路对抗评审（合规红线 / 定位漂移 / 现实可行性）
> 日期：2026-07-01 起排期｜来源反馈：dexter.dai（已登录，页面 /）

## 一、结论速览（TL;DR）

| 反馈 | 做/不做 | 一句话怎么做 |
|---|---|---|
| **② 数据源 References** | **做（立即，P0）** | 砍到最小：一个静态 `/methodology` 页 + 页脚统一入口 + 粗粒度"截至最近交易日"，复用"为什么动"已有来源范式，同步纠口径（联动有效率/关联/非确认）。零红线、信任地板正中靶心。 |
| **① 对话功能** | **重构后做（后置，P1/P2）；BYOK 不做；对个股解读收费在法务放行前不做** | 只做「锚点对话」——从卡片/个股页/简报的"追问"进入，范围锁死"解释已发生的、你自己的票"，绝不预测/荐股。用户原话"接模型进去"= 不照做（交出护栏=脱离定位）；"月度订阅"= 解耦后置，且付费只能锁额度不能锁个股解读。 |

**一句话主线**：这两条反馈是一套信任系统——References 是信任地板（先做、快赢、零风险），对话是搭子人格的载体（只有站在来源之上、锚定用户自己的票时才是"搭子"，脱离即退化成"选股机器人"）。**先夯地板，再谨慎长人格。**

## 二、反馈②：数据源 References（P0，先做）

### 2.1 定位与价值
- **它是什么**：信任地板（路线图 WS0）的正中靶心。用户信任一个搭子，是因为它会告诉你"这话我从哪听来的"。
- **零红线**：透明化 + "非确认"标注本身就是保守合规叙事的加固，不涉及任何投资建议风险。
- **用户主动提这个需求，本身就是对"盯盘搭子/保守合规"定位的正向验证。**

### 2.2 MVP 范围（对抗评审已把 S 砍到不膨胀）
**做：**
1. **新建静态页 `/methodology`「数据来源与方法」**（TTFB 友好，纯静态）：
   - 行情/实时 = 新浪；市值/概念/行业 = Tushare；新闻 = 博查 Bocha（带外链）；产业链关系 = **研究框架梳理·非确认关系**；战绩 = **联动有效率（历史复盘）**，收盘回填的透明规则。
2. **页脚 + 关键页面统一入口**，锚点跳转到 `/methodology` 对应段。
3. **粗粒度"截至"时间**：静态内容写"截至最近交易日 / 今日收盘"；仅实时行情显示时刻，且取自 `quotesAsOf` 真值。

**不做（避免 S 膨胀成 M，避开验收期回归风险）：**
- ❌ 不逐卡堆来源角标/logo（视觉噪音 + 给静态页增加 hydration，违背"能静态就不动态"）。
- ❌ 不把散落 15+ 组件的免责/来源文案在本次一起收敛到 constants（另立独立 hygiene 任务后置）。

### 2.3 实现思路（复用现有，不建新数据源）
- 直接复用"为什么动"的 `WhyData`（`sourceUrl/Title/Summary/Site/asOf`）+ `WhyCache` + 站内/外链弹窗范式，`/methodology` 是"把已有出处集中讲清楚"，不是从零建。

### 2.4 与"为什么动"的一致性
- 保持同一套来源展示语言（站内弹窗 / 外链），来源为空沿用 `why.ts` **零幻觉铁律：无材料不显示、绝不伪造来源**。
- **区分"检索到的事实来源"与"搭子的解读"**：AI 输出旁的来源不得为 AI 主观观点背书（红队定位漂移点）。

### 2.5 收益
信任地板 + 加固保守合规 + 直接回应用户 + 为日后对话提供"可核实来源"基础。ROI 极高、风险几乎为零。

### 2.6 放行前提（合规红队硬要求，做 References 时必须一并纠口径）
- `/methodology` 与所有用户可见处：**只讲"联动有效率（历史复盘）"，禁止出现"准确率/命中率"**。
- **用户可见的"受益"措辞改"关联"**（代码存量如 admin/outcomes/backtest 处一并核查）。
- **产业链关系的"研究框架梳理·非确认关系"标注须与关系内容同等显著**（不折叠、不弱化），与可外链核实的博查/Tushare 来源在视觉与措辞上分层，不得渲染成"受益关系图谱"或带因果箭头。
- 发页前核对新浪/Tushare 等**第三方数据署名的展示/使用条款**（具名来源可能触碰 ToS，署名反而可能制造法律暴露）。

## 三、反馈①：对话功能（重构后做，后置）

### 3.1 定位契合的形态：只做「锚点对话」

**它是什么：**
- 从"为什么动"卡片 / 个股页 / 简报条目的**"追问"**进入的对话，上下文永远绑定一个已存在的主体（这条异动 / 这只票）。
- 范围锁死在**解释已发生的与现在**：发生了什么 / 来龙去脉 / 市场在关心什么 / 这条对你的票意味着什么 / 风险点在哪 / 历史联动复盘。
- 答案接地于站内可核实材料（自选 + 记账 + 为什么动博查来源 + Tushare 事实），**无材料明确说"没有可核实依据"**。

**它不是什么（硬边界）：**
- ❌ 不是全站空白/悬浮对话框（开放框必被当成"股票版 ChatGPT"，滑成选股机器人）。
- ❌ 不是选股神谕：绝不答"该不该买/会不会涨/买哪只/目标价/加不加仓/还能不能上车"。
- ❌ 不是通用问答：不承诺"什么都能聊"。
- ❌ 不做"接任意模型/自带 Key"（见 3.3）。

### 3.2 合规护栏（chat 是全站红线暴露面最大处，护栏是核心工作量，非附属）

**硬红线清单（任何一条命中即改造，否则不上线）：**
1. **不得复用 explain 的 SYS 当护栏**——它只禁操作动词（买入/卖出/抄底/加仓/清仓），却鼓励"方向必须对/机会还是陷阱/超跌/该盯什么信号"，在自由多轮语境下直接输出预测涨跌+择时建议。**必须另写更严的对话专用 system prompt**：显式增禁"预测涨跌/会不会涨/目标价/点位/该不该买/买哪只/加不加仓"，删除一切前瞻+时机措辞。
2. **新建输入侧意图分类护栏**（不能复用 explain 的输出侧过滤）：红线问句 → 改写重定向而非直答。正则守不住（"上车/能进吗/跌到位没/谐音/错别字"），须以**接地 only（无可核实来源不作答）为主干 + system prompt 硬约束**，LLM 分类为辅并计入延迟/成本预算。
3. **每轮重锚 + 逐轮意图分类**：入口锚点只保第一轮，多轮里用户会 pivot（从"这票为什么动"滑到"那我该买吗"），必须每轮重新接地、话题硬锁。
4. **输出侧后置过滤**：对生成文本扫描操作动词 + 预测/目标价/涨跌断言词，命中即拦截或降级。
5. **免责在应用层强制追加**（不依赖模型自带）：每条 assistant 消息渲染即附 `DISCLAIMER`，**复制/分享/截图导出的文本也带上**；验证多轮、流式中断、复制三条路径下免责均在。
6. **口径**："受益"→"关联"；产业链"研究框架梳理·非确认"；战绩"联动有效率"不喊准确率。
7. **红线回归用例**（诱导选股/预测/目标价/换着说）纳入 `scripts/smoke.sh`，**拦不住即不上**。

**拒答话术（改写重定向，落点收窄为纯回溯/解释，移除一切"该盯什么信号/企稳/龙头脸色"等择时暗示）：**
> "我不替你判断买卖，但能帮你把这只票最近发生了什么、市场在关心什么、这次异动的来龙去脉讲清楚，你自己拿主意。"

**免责统一话术：** 每条输出末尾"以上为信息整理，不构成投资建议"。

### 3.3 "接模型进去"的解读与取舍：BYOK ❌ vs 托管 ✅

- **用户原话是投射 ChatGPT 范式，真实 JOB 是"用聊天的方式问一个懂我票的搭子"，不是真要选模型。**
- **BYOK（自带 Key）判否决**：① A股散户基本无 API key（伪需求）；② 用户自带模型输出仍在 StockTell 品牌下渲染，**零合规豁免**，可被诱导荐股后截图归责本站；③ 无法接地到自选/为什么动/Tushare，直接从盯盘搭子漂移成通用 LLM 前端。
- **取平台托管 + 我方策展 + 护栏的解释型对话**。话术="模型对你透明，不用操心接哪个"。

### 3.4 商业化（订阅分层 + 定价 + 免费付费边界）

**⚠️ 阻断级前提（法务另立项，先过法务再排期）：** 对特定证券作出分析/评价/解读并**收费**，境内极易被认定为"证券投资咨询业务"（需持牌），无牌收费诊股类 App 已有查处先例。**收费本身是监管定性触发点，而非话术层问题。**

- **在取得法律意见明确"不构成证券投资咨询经营"之前，不得对"个性化证券对话解读"收费。**
- **付费墙铁律（红队定位漂移硬红线）**：只锁与个股分析**解耦**的基础设施能力（更高对话额度 / 更快响应 / 去广告 / 导出）；**References / 数据透明 / 为什么动 / 雷区提醒 / 基础战绩等信任地板功能永久免费**——锁信任功能即反噬定位。
- **定价假设（待法务放行后验证）**：锚在"行情工具/陪伴会员"而非"荐股会员"，首发验证价 **19.9 元/月**（年付 168 打七折做锚）；叙事 = "更多对话额度 / AI盯盘搭子会员"，**绝不与收益、战绩、涨跌挂钩**；对外文案统一"信息服务/工具会员"，产品与合同条款禁止出现"诊股/分析个股/解读该不该盯"。
- **支付**：大陆刚需微信支付/支付宝（Stripe 不适用）。**先用兑换码/收款码人工开通验证付费意愿，再投正式支付基建。**

### 3.5 技术方案与成本

- **复用（约摊薄最容易的 20%）**：`getLLMFor('fast')` 流式管线、explain 的 `ReadableStream`、上下文拼装（fetchQuotes + STOCK_MAP + relations）、mid-stream 降级兜底、主/兜底切换 + Feishu 告警。
- **纯新建（真正难点与成本中心）**：对话专用严格 SYS、输入侧意图分类、输出侧后置过滤、**DB 级每用户每日配额**（现 `rate-limit.ts` 是内存尽力而为、多实例不一致，不足以护 token）、每分钟限流 + 输入长度上限 + 单用户并发单流、多轮上下文截断控 token、全量留痕。
- **上下文只做 prompt 拼装，禁止引入向量检索/embedding**（"RAG"叫法易诱发过度设计）。
- **区域**：`preferredRegion='hkg1'` 对齐博查/LLM，缓解大陆 TTFB。
- **成本模型**：多轮 token = 轮数×历史×用户，远高于一次性 explain；**配额是上线阻断项、非二期优化**，缺配额不发。

### 3.6 风险

| 风险 | 处置 |
|---|---|
| 自由对话诱发红线（最高危） | 另写严格 SYS + 输入分类 + 输出过滤 + 接地 only + 红线回归用例 |
| **无牌收费个股解读 = 无资质证券投资咨询（法律定性）** | 法务放行前不收费；收费能力与个股解读解耦 |
| **持仓/记账（敏感金融信息）经第三方 LLM + 博查香港区 = 敏感个人信息出境/PIPL** | 单独告知同意 + 出境合规；写入隐私政策（含对话保留期、退出） |
| LLM 容量/延迟：当前跑 DeepSeek 兜底 = 降级链路 | **先稳住主 provider，再上这个成倍放大 LLM 负载的常热动态功能** |
| 与移动端 UX 验收稳定窗口冲突 | 排到稳定窗口之后 |
| 投资者适当性/风险揭示义务 | 与法务一并评估 |

## 五、成功指标

**References（P0）**
- `/methodology` 页访问率 / 从数据展示处点击"来源"入口的点击率。
- 定性：反馈中"数据来源存疑/像编的"类质疑下降。
- 口径合规：全站用户可见处"准确率/命中率/受益"零残留（发布验收清单）。

**锚点对话（P1）**
- 从卡片/个股页/简报"追问"进入的发起率、人均多轮轮次、次日/7 日回访（"懂用户+沉淀关系"验证）。
- **合规护栏拦截率**：红线诱导用例（含换着说/谐音）拦截 100%（smoke 回归通过率）。
- 单位对话 token 成本 / 日配额触达率（成本可控性）。
- 接地率：带可核实来源的回答占比；"无依据"如实回答占比（零幻觉）。

**订阅（P2，法务放行后）**
- 兑换码激活数 → 付费转化率 → 次月续费率（先质量后流量，先验证意愿再谈规模）。

## 六、明确不做 / 暂缓（红队"阻断"项落地）

| 被判 | 结论 | 原因 |
|---|---|---|
| **BYOK / 自带模型 / "接任意模型进去"** | **不做** | 交出护栏=脱离定位；输出仍在本站品牌下渲染=零合规豁免、可被诱导荐股后归责本站；散户无 key=伪需求。 |
| **全站空白/悬浮对话框** | **不做，改锚点入口** | 开放框必被问"该不该买/会不会涨"，全站红线暴露面最大，把搭子拉成选股机器人。 |
| **复用 explain SYS 当 chat 护栏** | **不做，另写更严 SYS + 输入分类 + 输出过滤** | explain SYS 只禁操作动词却鼓励方向判断/择时，自由多轮下直接踩预测涨跌红线。 |
| **改写重定向到"该盯什么信号/企稳/龙头脸色"** | **不做，落点收窄为纯回溯/解释** | 择时提示同属投资建议（择时建议），只是从明牌荐股滑到暗示择时。 |
| **对"个性化证券对话解读"收费** | **法务放行前不做** | 极易被定性为无资质"证券投资咨询经营"，收费本身即触发点。 |
| **把 References/为什么动/雷区/基础战绩锁进付费墙** | **永久不做（硬红线）** | 锁信任功能直接反噬定位。 |
| **月度订阅与 chat 捆绑上线 / 提前改 schema 加 plan 字段** | **暂缓，解耦后置** | 未验证功能提前建支付基建+改库（走 init-db 运维成本），范围暴涨；先兑换码验证意愿。 |
| **References 逐卡堆来源角标 / 本次收敛 15+ 文件到 constants / 分钟级 asOf** | **暂缓/砍掉** | 视觉噪音 + 违背"能静态就不动态" + 移动端刚改完扩大回归面；过期时间戳反噬信任。 |
| **敏感意图监控看板** | **暂缓** | 单用户反馈阶段属过早基建；先全量留痕落库即可，不建监控面板。 |
| **在 DeepSeek 兜底链路上上线 chat** | **暂缓至主 provider 稳定 + 过验收窗口** | 在流沙上盖楼；chat 是常热动态+LLM 密集功能，与降级链路和稳定窗口直接冲突。 |

**补齐三项被全员遗漏的合规维度（对话立项时必须覆盖）**：① 敏感金融信息出境/PIPL 单独告知同意；② 投资者适当性/风险揭示义务评估；③ 含持仓的对话留痕在隐私政策写明保存/退出/保留期。
