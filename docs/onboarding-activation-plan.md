# StockTell 新用户首跑激活方案

> 产出方式:多智能体工作流(小白实走核验 → 激活/漏斗/文案3视角 → 对抗评审 → 综合)。日期 2026-06-30。



## 1. TL;DR

**现状诊断:** 当前首跑把激活成败押在一个偶然条件上——「当天恰好有命中我自选的已发布简报」;只加 1 只票的小白大概率走进 5B「没异动 我盯着呢」黑洞,被 toast 抬高的期望当场落空就走人,且不回来——首跑激活率保守估计仅 20-30%。

**推荐优先做的 3 件事(低成本、不加摩擦、能快验):**
1. **安静票当场给一句人话**(把 5B 黑洞从"没异动"改成"每只票一行:名称+涨跌%+它靠什么吃饭")——把 70% 走进黑洞的人当场救回,数据全有、不依赖简报、合规最干净。
2. **一键加示例票**(搜索框上方一排 chips,结构性选票,内联不加新屏)——把"全靠打字"的冷启动门槛降到零打字。
3. **toast 从期票改兑现 + 埋 4 个核心事件**(landing/hit/miss/D1回访)——消除期望落差,并用真实数据验证前两件是否有效,而不是继续拍脑袋。

## 2. aha 定义与最短路径

### aha 是什么(可埋点)
> 用户在首跑会话内,针对"自己刚加的票",读到至少一条"说人话、与这只票直接相关、当下即时"的产业链解读,并看懂了它。

代码里这个时刻 = `BriefingFeed`「和我相关」区出现 `MorningBrief`(☀️你的今日早报)+ 命中自选的简报卡。

### 当前有多少人走不到、为什么
能走到 aha 要**同时**满足三个前提(任一断裂即出局):
1. 肯打字加上至少 1 只票 —— 无一键加,小白这里掉一批,且常只加成 1 只;
2. 加的票在池内(约 122 只 A 股 + 美股锚点)—— 冷门票直接出局;
3. 当天恰好有"已发布且命中这只票"的简报 —— **这是最硬的偶然条件**,全局每日生成、只点名少数票。

叠下来:愿意加票的小白里只加 1 只是常态,单只票当日命中简报概率乐观也就两三成。**保守估计首跑当天真正到达 aha 的新用户只有约 20-30%,很可能更低。** 大多数人(尤其只加一只、又逢安静日)走进 5B 的 `QuietMorningBrief`。

### 关键判断
加了一只"今天没异动"票的人会走——而且是被自己刚加票的动作 + toast 承诺抬高期望后摔下来,体感比"没加票直接看大盘"还差。`QuietMorningBrief` 的"我盯着呢👀"兑现的是**未来价值/留存承诺**,而首跑用户此刻要的是**当下就让我看懂一件事**的即时价值——两者错配。盯盘搭子的留存逻辑成立,但它把激活的赌注押在了"用户愿意空手回来第二天"上,对没有信任基础的第一批用户太脆弱。

### 最短路径(Time-to-Value 目标 < 30 秒)
```
落地(首屏一行承诺) → 一键加示例票(0 打字) → 加完立刻看到「你的票今天怎么看」
→ 命中日:读到个性化早报 = aha;安静日:读到"它靠什么吃饭+今天怎么走" = aha(保底版)
```
**核心原则:aha 必须不依赖"当天有命中简报"。** 命中是豪华版,安静日的"它在链条里靠谁吃饭 + 今天怎么走"是保底版——两条都满足 aha 定义。

## 3. 当前首跑关键堵点(按伤害排序,来自小白实走)

1. **【头号杀手】"今天没异动" = 激活黑洞。** 加完票被 toast 承诺"会给你看相关动态",若当天没命中,看到的是 `QuietMorningBrief`"没异动 我盯着呢"。只加 1 只(尤其加了安静票)的小白当场判定"我专门加了票结果啥也没有"→ 走人 + 不回来。落差最大、最致命。

2. **aha 依赖偶然条件。** 命中与否由客户端 `isMine`(`triggerCode` 命中 或 `beneficiaries` 命中)判定,而简报全局每日生成、只覆盖少数被点名的票。单只票当日命中率本就低。

3. **加自选全靠打字,无一键加、无 payoff 预览。** 完全小白第二只不熟的票(记不全名字、不懂 6 位代码)直接放弃,常只加成 1 只 —— 进一步压低当日命中率,与上一条叠加恶化。

4. **首屏第一张价值卡是大盘黑话。**「AI链今日情绪」(费半/主力净流入/AI链温度/隔夜美股)对完全小白是噪音不是钩子,且与"我的票"零关联,没立刻兑现 slogan"告诉我怎么想",削弱继续下滑动力。

5. **深读一点就弹登录。** 免费早报可看是对的,但每张卡的"🔍深读"在用户还没体会价值前要求登录,部分人误以为付费/嫌烦而退。(伤害较轻,且时机本身在 aha 之后,属可接受门槛。)

6. **池外票负面观感。** 用户那只"不熟的票"若不在 AI 链(很常见),搜出来是"暂未纳入/没找到",小白得出"这站连我的票都不收"的负面结论,信任受损。(次要。)

## 4. 改造方案 P0 / P1

> 总原则(吸收评审):**真正抬激活的只有 2 件,其余多为"为引导加引导"或在最该顺滑处加屏。** 本轮 P0/P1 只做低成本、不加摩擦、能快验的;被否的过度设计统一进 §6 不做清单。

### P0-1 ★ 安静票当场给一句人话(堵住头号黑洞)
- **问题:** `BriefingFeed.tsx:68-72` 没命中分支只给 `QuietMorningBrief`"没异动 我盯着呢👀" + `WatchOverview` 涨跌行。这是未来价值承诺,与首跑用户要的即时价值错配,当场流失。
- **改法:** 把没命中分支从"只说没异动"升级为"每只票各给**一句**即时人话"。料全部已有、不依赖当日简报:每票的 `positioning`(它靠什么吃饭)来自 `STOCKS`(静态),当日 `change` 来自 `/api/quotes`(`WatchOverview` 已在拉)。**吸收评审:一张卡只留一句人话(positioning),"为什么动"有料才显示、无料不编;"它盯谁/上游看英伟达/博通"砍掉或折叠——否则只是把"没异动"黑洞换成"读不懂"黑洞。**
- **落在哪个真实组件:** 新增 `src/components/QuietWatchCard.tsx`,替换 `BriefingFeed.tsx:68-72`(`QuietMorningBrief`)分支的内容部分;数据源 `src/data/stocks.ts`(positioning)+ `WatchOverview`/`/api/quotes`(change);"为什么动"可选复用 `/api/briefing/why`(`WhyProvider` 已按 code 批量取,安静票同样可查,后端没料就不显示)。`QuietMorningBrief` 的"我盯着呢👀"陪伴口吻保留为卡片底部一行,不再是唯一内容。
- **交互/文案示例:**
  ```
  ☀️ 你的今日早报 · 今天你的票没踩 AI 产业链的雷,各说一句 👇

  中际旭创  +1.2%
  它靠给海外云厂(微软/Meta)的 AI 服务器供高速光模块吃饭。
  [若 why 有料] 今天跟着算力链情绪偏暖；[无料] 今日无单独消息,跟随板块。

  （同款卡逐只渲染；只加 1 只也成立）  …明天我接着盯 👀
  ```
- **预期效果:** 把约 70% 走进 5B 的人从"啥也没有"救成"当场看懂自己票在链条里的位置" = 一次保底 aha。本方案投入产出比最高的一点。
- **改动量:** 中。1 个新组件 + 改 1 处分支;复用现成数据与 API,无后端新接口。

### P0-2 ★ 一键加示例票(降冷启动门槛)
- **问题:** `QuickAddWatch.tsx` 只有搜索框,完全小白第二只直接放弃、常只加 1 只 → 压低命中率。
- **改法:** 搜索框上方加一排「示例票」chips,点一下 = `wl.toggle(code)`(0 打字),可多选鼓励加 2-3 只。**吸收评审的两条关键约束:**
  - **(a) 这条的灵魂不是"省打字",而是选票逻辑。** 选票理由对外用**结构性**口径(链条核心节点 / 大家都在看),**不**把"按当日命中择时"作为对外可感知的逻辑——后者动态、择时、和当日行情挂钩,比固定列表更像"今日推荐池",合规风险最高。用结构性理由选公认锚点(中际旭创/工业富联/寒武纪/海光信息等),它们命中率本来就高:既拿激活收益,又不落"择时荐股"口实。
  - **(b) 无自选用户首屏新增元素 ≤1 个**——就是这排 chips,内联在搜索框上方,不单列 Hero、不加弹窗、不阻断。
- **落在哪个真实组件:** `src/components/QuickAddWatch.tsx`(搜索框 `:78-87` 上方加 chips);仍走 `useWatchlist`(localStorage,**不强制登录**,与现状完全一致)。
- **交互/文案示例(中性化合规处理):**
  ```
  不知道加哪只?点一下就行,随时能删 👇
  [中际旭创] [工业富联] [寒武纪] [海光信息] …   ← 大家都在看的
  记得自己的票?也可以直接搜 ↓
  示例仅为方便体验,不构成任何建议  ← 极小灰字,合规兜底
  ```
  **合规红线:** 禁用"推荐/精选/值得关注/潜力/强势/热门龙头";只用"大家都在看的 / 这几天常被提起"这类客观曝光度描述;配免责小字。
- **预期效果:** 提高加票数 → 抬当日命中率 + 抬 P0-1 安静卡覆盖面。
- **改动量:** 小到中。1 个组件内联改动 + 给 `add_watchlist` 加 `kind:'example'|'search'`、`count_after` 两个 prop。

### P1-1 toast 从期票改兑现(消除落差,零成本)
- **问题:** `useWatchlist.ts:110` `ADDED_MSG`"已加入自选 · 首页「和我相关」会给你看相关动态"是期票,把期望抬满却把兑现推到"将来某天有动态",直接喂给 5B 落差。
- **改法:** 改成兑现口径(配合 P0-1,安静日也有内容,承诺不再落空)。
- **落在哪个真实组件:** `src/components/useWatchlist.ts:110`。
- **文案示例:**
  > 旧:`已加入自选 · 首页「和我相关」会给你看相关动态`
  > 新:`已加 · 下拉「和我相关」,这就给你看它今天怎么看 👇`
- **预期效果:** 消除期望-现实落差。
- **改动量:** 极小,一行字。

### P1-2 首屏一行承诺条(兑现 slogan,不重排黑话卡)
- **问题:** `page.tsx:75` 首屏第一张是 `ChainSentiment` 大盘黑话卡,对小白没立刻兑现 slogan,削弱下滑动力。
- **改法:** **吸收评审——不重排/不折叠 ChainSentiment**(那会引入两套布局分支、维护与回归风险大,收益只是少看一眼)。**轻量版:仅对无自选用户,在 H1 副标题下、`ChainSentiment` 之上加一行文案承诺条**,把"加你的票"前置到第一眼。可并进 P0-2 chips 的标题,不必单列组件。
- **落在哪个真实组件:** `src/app/page.tsx`(H1 副标题与 `:75` ChainSentiment 之间);条件 `wl.ready && wl.codes.size===0`,有自选用户无感。
- **文案示例:**
  > 「我不懂产业链,你告诉我怎么想」—— 加上你手里的票 👇 我每天用一句人话告诉你:今天 AI 产业链的动静,跟你这只票有什么关系。
- **预期效果:** 提高下滑/加票转化。
- **改动量:** 小。一段条件渲染文案,不动 ChainSentiment。

## 5. 激活漏斗与指标

### 北极星(NSM)
**首跑激活用户数** = 单个新用户首跑会话内同时满足 (a) 至少加 1 只自选 且 (b) 看到与自己票相关的内容(命中早报/卡 **或** P0-1 安静卡)。把"加了票"和"拿到 aha"绑在一起,正对齐产品定位,且不靠喊收益(合规)。
辅助北极星(留存):**次日带自选回访的用户数**(D1)。

### 现有埋点盘点(已核验 src)
已在用:`add_watchlist{kind}`、`signup`/`login{method}`、`bind_push`、`deep_read`、`overnight_radar_view`、`submit_feedback`、`pwa_installed`。
**缺口:** 落地曝光、aha 命中/落空、次日回访全无埋点——激活漏斗中段是盲区。

### 早期只埋这 4 个事件(吸收评审:其余全部缓做)
> 理由:第一批用户、样本量撑不起 8 个转化率的统计意义;没历史数据的基线目标就是拍脑袋。先用最小集两周跑出真实分布,再补其余。

| 事件 | 新增/改造 | 关键 props | 作用 |
|---|---|---|---|
| `landing_view` | 新增 | `source`, `hasWatchlist` | 漏斗分母 |
| `add_watchlist` | 改造(加 prop) | `kind:'example'\|'search'`, `count_after` | 验证一键加是否有用、用户加几只 |
| `relation_hit` / `relation_miss` | 新增 | `watch_count`, `has_morning_brief` | **当前唯一真盲区,验证 P0 该不该做的唯一证据** |
| `returning_visit` | 新增 | `days_since_first`, `hasWatchlist` | 留存唯一硬指标 |

`relation_hit`/`miss` 应在 `BriefingFeed` 水合算出 `isMine`/MorningBrief 模式那一刻埋,每会话一次。埋点落点:`src/lib/analytics.ts`、`useWatchlist.ts:108-110`、`BriefingFeed.tsx` 分支、`src/app/layout.tsx`(landing/首日时间戳)。

### 早期基线靶(建基线用,非 KPI 承诺,两周后按真实中位数 ±一档校准)
- 首跑加票率(`add_watchlist(first_add)` / `landing_view`):粗靶 12-25%
- 到达 aha 率(`relation_hit` / 加票用户):粗靶 20-35%(现状很可能偏低端)
- 首跑激活率(NSM / `landing_view`):粗靶 5-12%(0.18×0.30≈5.4%,自洽)
- 次日留存(带自选)(D1回访 / 加票用户):粗靶 15-30%

### 用数据判断"安静票黑洞"严重度(决策规则)
- 规模:落空率 = `relation_miss` / (`hit`+`miss`);按 `watch_count` 切分,看"只加一只"的惩罚幅度。
- 因果:对比 hit 组 vs miss 组的 **D1 留存差**——这是最硬证据。
- 一句话判定:**`relation_miss` 占比 >50% 且 miss 组 D1 留存 < hit 组 0.6 倍 → P0 激活阻断,优先于一切新功能;若占比高但两组次留接近 → 降 P1(陪伴留存叙事在起作用,可缓)。**

## 6. 不做 / 慎做清单(吸收评审)

**明确不做(本轮):**
- **加前 payoff demo 静态卡。** 典型"为引导加引导":在最该让用户动手的位置插一屏"先读说明书",而一键加点下去 1 秒就出真卡——demo 卡存在的那 1 秒纯属多余。真正的预览是"点一键加→立刻出真卡"这个动作本身。砍。
- **ChainSentiment 重排 / 条件折叠 / 改标题。** 给无自选用户做两套布局分支,维护与回归风险不小,收益只是少看一眼黑话卡。用 P1-2 一行文案条压在上面即可,不动它的位置。
- **单列 FirstRunHero 组件。** 与 chips、demo 叠在首屏会变成"弹窗墙",破坏"打开即用"。Hero 文案并进 chips 标题,不单列。
- **QuietWatchCard 里的"它盯谁:上游看英伟达/博通"。** 对小白又是产业链术语,会把"没异动"黑洞换成"读不懂"黑洞。一张卡只留一句 positioning 人话。
- **深读登录框文案优化。** 过度优化;深读门槛在 aha 之后,时机本就合理。
- **早期就铺满 8 个转化率 + 三层严重度框架 + 全套基线目标区间。** 把成熟产品分析体系提前搬进来,早期样本撑不起统计意义。只做 §5 那 4 个事件。

**慎做 / 延后(数据出来或资源允许再说):**
- 池外票留资钩子("你的其它票正在扩 → 留邮箱通知")——救信任的次要点,延后。
- PWA / 邮件订阅钩子的全套时机设计——方向对(必须 aha 之后、可关、不阻断、不复弹),但非本轮激活核心,延后。
- 示例票池"按当日命中动态择时"——**慎做且不对外暴露**:动态择时+挂钩当日行情,合规观感最像"今日推荐池"。本轮用结构性选票(顺带命中率本就高);若未来要做,必须配齐中性化措辞+免责,且选票对外理由始终是结构性而非择时。
