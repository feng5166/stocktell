### 1.4 因果链(Hops,已区分主链/分支 + 补证伪支)

> **结构声明(反换皮关键)**:本链**真正层层递进的推理主链**是 Hop1→Hop2(事件→token 放量→capex 上修);Hop3–Hop8 是**由「GPU/数据中心↑」这一节点扇出的并行资产映射分支**,对本次具体事件的**边际敏感度低于 Hop1–2**,不以跳数堆砌伪装深度。

#### 主推理链(递进)

**Hop1 ·【关键假设·全链总闸】模型能力跃迁 / 推理成本↓ → AI 调用量(token 消耗)↑**
- 逻辑:单位推理成本下降 + 能力提升,推动应用侧调用频次与场景扩张(杰文斯悖论),总 token 消耗不降反升。**边界条件**:在需求弹性不足/企业预算约束/场景不成熟时,并非线性放大。
- **【新增·空头读法/证伪条件】** 「推理成本↓」对硬件本是**多空分岔**:杰文斯放量(多)vs 单位硬件需求下降、硬件端毛利承压(空)。**若杰文斯不成立(降本主要转化为省钱而非放量),则本链方向反转。此假设成立与否是整条链置信度的总闸。**
- 证据类型:官方定价页 / 产业数据 / 财报电话会纪要。
- 证据举例:模型厂商官方定价页(每百万 token 价格逐代下降原文)+ 头部 API/云平台月度 token 处理量增速;或财报电话会「推理需求占比提升」原话。**引具体数字前必须回原始来源核对,不预填。**
- 置信度:**中高**(受杰文斯边界条件下调,与自曝口径一致)。

**Hop2 · 调用量↑ → 数据中心算力 / GPU 采购(capex)↑**
- 逻辑:推理负载增长需要更多加速卡与集群,北美云厂上修资本开支承接需求。
- 证据类型:财报 / 官方 capex 指引。
- 证据举例:微软/谷歌/Meta/亚马逊季报 capex 指引与「AI 及数据中心投入」口径;英伟达数据中心分部收入与下季指引原文。
- 置信度:高。

#### 并行资产映射分支(由 Hop2「GPU/数据中心↑」扇出;对本次事件边际敏感度较低)

**Hop3 · GPU 需求↑ → HBM 高带宽存储需求↑**
- 逻辑:每颗 AI GPU 强制配套多颗 HBM,近似线性绑定,HBM 持续供不应求。
- 证据举例:美光、SK 海力士财报「HBM 产能已售罄/年度产能预定」与扩产计划;第三方 HBM 位元需求预测。
- 置信度:高。

**Hop4 · GPU/HBM 需求↑ → 先进封装(CoWoS/2.5D)产能瓶颈↑**
- 逻辑:HBM 与逻辑 die 集成依赖 CoWoS,封装产能而非晶圆前道往往是算力供给实际卡点。
- 证据举例:台积电法说会 CoWoS 扩产表述;OSAT(日月光、Amkor)先进封装稼动率与订单能见度。
- 置信度:高。

**Hop5 · GPU 功耗/机柜密度↑ → 液冷/温控渗透率↑**
- 逻辑:单机柜功率跃升至风冷极限之上,液冷由可选变刚需。**渗透节奏与建设周期存在不确定性,不宜线性外推。**
- 证据举例:英伟达 GB 系列机柜整机功率官方 datasheet;Vertiv 等液冷方案渗透率/在手订单口径。
- 置信度:中。

**Hop6 · 算力集群规模↑(scale-out)→ 光模块/高速互连需求↑**
- 逻辑:万卡/十万卡集群横向扩展,GPU:光模块配比抬升,速率向 800G/1.6T 迭代。
- 证据举例:中际旭创、新易盛季报高速率光模块收入占比与北美客户结构;第三方速率迭代路线图。
- 置信度:高。

**Hop7 · 机柜内 scale-up 互连 → 高速铜连接需求↑**
- 逻辑:机柜内短距高速互连以铜缆为主,GB 系统铜背板放量;但光/铜边界随距离与速率变化,**存在替代博弈,需警惕拥挤交易与路线切换风险**。
- 证据举例:英伟达 NVLink 铜背板官方规格;高速铜缆/AEC 客户验证与订单进展纪要。
- 置信度:中。

**Hop8 · 数据中心规模↑ → 电力供给/供配电(HVDC/变压器/发电)约束↑**
- 逻辑:算力集群总功率激增,电力从「成本项」升级为「能否上电」的核心约束。**传导到 A 股电力设备属较间接的一层。**
- 证据举例:云厂/电网数据中心用电增速公开表述;Vertiv、Eaton 供配电订单积压;当地电力规划/运营商算力网络投资。
- 置信度:中。

- **price-in 未校验**:A 股光模块龙头是否已被资金充分定价,需结合当日行情与资金面再核。

## 2. 它凭什么不是新闻聚合(用本产物自证)

资讯聚合(和东财/同花顺「AI 概念板块」)给你的是:**一条新闻 + 一串相关个股清单**。本产物多给了三层「判断」,而判断是清单给不了的:

1. **传导层级排序,不是罗列**:同样面对「英伟达财报好」,聚合告诉你「AI 板块相关」;本产物告诉你**光互连(直接绑北美云厂订单)的传导链比国产算力芯片(情绪映射)完整**,并说清凭什么这么排。
2. **个股关系分级(直接/间接/情绪映射)**:聚合把中际旭创、海光信息、东方电气并列在同一个「AI 概念」里;本产物明确区分**直接供货 / 上游配套 / 国产替代情绪 / 能源宏大叙事**——把「香农芯创是代理不是自研 HBM」「澜起是 DDR5 不是 HBM」「东方电气本质是能源股」这些**聚合永远不会告诉你的证伪信息**摆到台面。
3. **逐跳置信度 + 诚实自曝 + 证伪支**:聚合从不说「这条逻辑可能反转」;本产物在 Hop1 明确标「杰文斯若不成立则全链反转」,并给每跳配证据类型与置信度。
4. **事件驱动的差异化**(反换皮内核):同一条链在「推理降本事件」与「训练扩容事件」下**必须产出不同的热力排序**——这是「因果推理」而非「按类别检索预制图谱」的分水岭,也是聚合/概念板块结构上做不到的。

> **诚实边界**:当前推理增量主要集中在 Hop1(杰文斯)+ Hop2(capex),Hop3–8 是并行映射分支。所以本产物「强于聚合」主要强在**排序 + 关系分级 + 置信度 + 事件敏感**,而非「8 跳深链」。这个边界必须对内说清,不对外吹「深度推理」。

## 3. 最小落地 spec(推理链详情页)

### 3.1 要建什么(新建)

### 3.3 明确不建什么(硬约束,防蓝图外溢)

## 4. 验收指标(怎么算内核被验证了)

### (A) 产物质量线(内部可判,发布前必须全绿——**任一不达标即 no-go**)

| 指标 | 阈值(成立) | 不成立 |
|---|---|---|
| 每跳可核实 | 100% hop 有明确证据类型,上线版每个引用数字均回原始来源核对 | 有 hop 无法给出可核来源 |
| 置信度诚实 | Hop1 标为「中高 + 证伪支」,弱映射(香农/东方电气/电连)如实降级 | 出现「全高置信」或掩盖弱项 |
| 零编造 | 无杜撰数字/URL/供货关系;「待验证」如实标注 | 出现任一未经核对的确定性数字 |
| 不踩荐股红线 | 无「利好/受益/确定性最高/估值/弹性/高位/主线/目标价/买卖」词;个股逐条挂非确认标签;有合规展示模式 | 命中任一红线词或缺标签 |
| 明显非聚合 | 同一事件类型 vs 另一事件类型能产出**不同热力排序**;关系分级完整 | 换事件出同一张图(=换皮) |

### (B) 用户价值信号(上线后 2–4 周观测)

| 信号 | 成立阈值 | 说明 |
|---|---|---|
| 看完整链比例 | 打开详情页用户中 **≥35%** 滚动/展开到因果链底部或映射表 | 证明「愿看完整链」而非只看标题 |
| 分享 | 出现**自然分享**(分享率 ≥3%,或绝对量达小样本可观测) | 证明「觉得值得传播」 |
| 据此加自选 | 从映射个股卡片跳转→加自选转化 **≥8%**(相对入口 PV) | 证明「据此行动」的价值信号 |
| 定性访谈 | 5–8 位目标用户,**≥半数**能说出「比看新闻/概念板块多知道了什么」且点到「排序/关系分级/证伪」任一 | 定性佐证内核被感知 |

> **综合判定**:(A) 全绿是上线闸门;(B) 中「看完整链 + 定性访谈」两项达标即视为**用户价值信号成立**(分享/加自选为加分项,小样本下不作为一票否决)。(A) 绿但 (B) 不成立 → 产物可产出但用户不买账,内核需重定位;(A) 都做不到 → 内核在「可产出性」上就不成立。

## 5. 合规护栏(措辞红线,落进产物与页面)

**阻断项(不做完不能上线)**
- 个股映射**逐条**挂标签:「研究框架梳理·非确认关系,不构成对该证券的推荐或买卖建议」;个股定位为**说明性示例**,只陈述业务位置、不排优劣受益。
- relation 字段注明:「传导层级(供货/配套距离),非受益程度或收益排序」。
- 提供**「仅展示产业环节、隐藏个股」合规展示模式**(默认或可切换)。

**必删/必改词(全站扫一遍)**
- 删「利好 / 受益 / 确定性最高 / 估值偏高 / 别按估值给估值 / 弹性大 / 高位接棒 / 题材 / 主线 / 逻辑最硬」。
- 「利好的是」→「与本类事件产业逻辑关联最紧的是」;「放量确定性最高」→「传导逻辑完整度最高(非上涨确定性)」;「逻辑最硬」→「证据链相对完整、传导最直接」。

**口径隔离(页面显式声明)**
- 热力图:「景气上行/关注上升/结构分化/景气承压 = 产业景气/关注度,非股价预测」。
- 排序:「传导层级/证据完整度强弱,非推荐顺序、非收益或涨幅排序」。

**免责补充**:「本工具不提供证券投资咨询服务,所列个股仅为产业链关联的说明性示例,不构成任何推荐。」

**事件替换合规清单(每次换真实 seed 必走)**
1. 引用须回原始来源核对数字;2. 个股只保留业务事实与关联层级,禁止任何前瞻收益/价格表述;3. 复用统一免责与「非确认」标签;4. 重扫必删词。

## 6. GO / NO-GO 与最大风险

### 结论:**GO**(限定最小可证)

**这条最小可证能证明什么**
- 结构化多跳因果链产物**能否被人工策展产出**、且**明显强于新闻聚合**(排序 + 关系分级 + 置信度 + 事件敏感)。
- 产物**能否合规呈现**(个股映射语义重构后是否守得住红线)。
- 用户**是否觉得有用**(愿看完整链 / 分享 / 据此加自选 / 定性认可)。

**这条最小可证不能证明什么(诚实说清,防误读)**
- ❌ **自动化/规模化未验证**:本阶段是「人工策展 + LLM 辅助」的**单条链**。它证明的是「产物值钱」,**不证明「能自动量产、边际成本可控」**——这是下一阶段(且只有本阶段成立才启动)的问题。
- ❌ **推理深度未证强**:真正递进的推理集中在 Hop1–2,不宣称「深链」。
- ❌ **不证明选股/收益**:产物是研究框架,不含胜率、不预测价格,验收也不看「涨没涨」。

### 最大风险(按优先级)

1. **换皮风险(战略头号)**:若链对「事件」不敏感,任何 AI 利好出同一张图,就退化成「检索预制产业链图谱」= 最像东财、内核不成立。**收口手段**:第 1.1 节「为什么是这个事件」必填 + 换事件测试(验收 A 的「明显非聚合」项)。
2. **合规阻断风险**:个股+代码绑定方向性利好是券商级红线。**收口手段**:第 5 节阻断项全部落地,否则不上线。
3. **产业硬伤风险**:澜起(DDR5≠HBM)、电连技术(题材≠间接)、沃尔核材(供货待验证)等归属/关系夸大——已在 1.5 修正,上线前需二次核收入结构。
4. **单人 + 并行维护的执行风险**:蓝图不得外溢(见 3.3 不做清单),严守单链单页。

**建议**:批准 5–8 人日投入,产物策展先行(它就是内核本身),页面复用现有资产;发布闸门 = 验收 (A) 全绿 + 合规阻断项清零;上线后 2–4 周看 (B),再决定是否进入「自动化可行性」的下一步论证。
