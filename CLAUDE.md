# StockTell · 项目内核(必读)

> 本文件是所有贡献者(技术 / 产品 / 美术,含 AI 助手)动手前的必读内核。第一条永远是产品的立身之本。

## 🧭 第一内核:核心竞争力 = 事件 → 产业链 → 个股影响

**StockTell 的核心竞争力 = 把「事件 → 产业链 → 个股影响」串成一条可核实的因果链。不是新闻聚合。**

- 别人给"发生了什么"(新闻/行情堆叠);我们给"这事顺着产业链传导下来、对你手里的票意味着什么"。
- **护城河** = 产业链关系图谱(谁供货谁/对标谁)+ 事件溯源 + 落到个股的因果解读 + 懂用户的票。新闻聚合/行情罗列是红海、无壁垒。

**三条硬规矩(做任何功能/交互/视觉前自问):**
1. **它强化这条链吗?** 让"事件如何沿产业链传导、最终影响哪只票"更清晰/可信/个性化。只是"多一条新闻/行情"→ 不做。
2. **落点永远是"对我的票意味着什么"**(用户自选/关注的票),不是泛泛资讯。
3. **可核实、不编造**:因果链每一环要有依据(来源/关系边/复盘数据),守保守合规。

**各角色落地:**
- **技术**:数据/检索都服务这条链——`chainEdges`(产业链边:方向+强弱+依据)、「为什么动」(事件→真实来源)、联动雷达(跨市场传导)、个股页(事件+产业链位置+对我的影响)。新数据/接口先问"是否喂养这条链"。
- **产品**:每个页面回答"这事对我的票意味着什么";信息架构围绕 事件→传导→个股,不是新闻时间流。
- **美术**:视觉让因果链一眼可读(方向/强弱/传导路径),不是新闻列表堆叠。

**⚠️ 反面(别做成):** 纯新闻 feed、纯行情看板、资讯聚合器、"什么都报但不解释对我的票的影响"。

## 其它约定
- 产品定位:A 股散户的 AI「盯盘搭子」(陪盯盘的搭子,不是荐股/喊单工具)。
- 合规口径**保守**:不喊单/不荐股/不预测涨跌;战绩讲"联动有效率"复盘、不喊准确率;"受益"→"关联";AI 输出挂"不构成投资建议";关系标"研究框架梳理·非确认"。
- 外部服务/环境变量:见 [`docs/EXTERNAL_SERVICES.md`](docs/EXTERNAL_SERVICES.md)(新增服务先查、接入后更新)。
