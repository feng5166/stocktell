# StockTell · 项目内核(必读)

> 本文件是所有贡献者(技术 / 产品 / 美术,含 AI 助手)动手前的必读内核。第一条永远是产品的立身之本。

## 🧭 第一内核:全球事件 → 本土资产的因果链推理工具

> **完整定稿见 [`docs/PRODUCT-CORE.md`](docs/PRODUCT-CORE.md)。后续讨论问题、开发任何新功能,先对照那份检查清单。**

**StockTell 不是股票软件、不是新闻聚合,是 AI-native 的「产业理解工具」**(长期愿景 = AI 时代的 Bloomberg)。第一阶段切口:**全球事件 → 产业链传导 → 本土资产映射。**

**产品公式(= 验收标准):** `事件输入 × 产业链图谱 × 多跳推理 × 资产映射 × 证据验证`
口语版:**从一件全球大事,推演出它会影响中国哪些产业链和股票,并给出每一跳依据。**

**核心动作 = 多跳推理(范式,牢记):**
```
OpenAI 发布 GPT-6 → 推理成本下降 → AI 调用量↑ → 算力/GPU 需求↑
  → HBM / 先进封装 / 液冷 / 光模块 / 电力设备升温
  → 国内映射:海光 / 长电 / 中际旭创 / 英维克 / 沃尔核材 …
```
单跳(英伟达涨→A股AI涨)太浅无壁垒;**多跳**才是护城河。要回答:①为什么重要 ②沿哪些环节传导 ③国内哪些票受影响 ④每跳依据 ⑤历史是否有效。

**交付三要素(每个场景尽量都给):** ①**一句话判断**(结论先行,不丢一堆数据)②**产业链热力**(哪些环节升温/降温/分化,早期结构化矩阵即可)③**可核实因果链**(每跳有依据+置信度,不伪装确定性)。

**三条铁律(硬边界):**
1. **绝不做新闻聚合** —— 只收事件不堆新闻;必须推演到"为什么重要/影响哪条链/传导到哪/国内映射/依据/历史是否有效"。
2. **绝不输出无依据结论** —— 每个判断能回溯到事件/数据/公告/财报/关系边/复盘;无强证据处标"推理假设/低置信度/待验证"。
3. **绝不做交易指令(= 定位铁律,非合规)** —— 不输出买/卖/目标价/涨跌预测;个股永远是"关系分级的说明性示例",不是"推荐买入"。只做产业链解释、影响映射、历史复盘、风险提示。内容/分享卡/文案/AI对话一起执行。

**合规姿态(2026-07-02):启动阶段合规从宽,产品定位不变。** 服务器海外+域名不备案(绕开 ICP/内容登记轴)→ 内容层放开、不为纯措辞畏手畏脚、按可接受风险快跑。但"不喊单/产业理解工具"是**定位与身份、不随合规松紧变**,故铁律3 永远守(喊单=掉回荐股红海+拆信任护城河)。offshore 不覆盖证监会轴的真触发点=收费/分享传播/个人 exposure;其中**订阅收费+个股解读**上线前仍要一纸法律意见(主体/律师另立项),其余启动期从宽。详见 [`docs/PRODUCT-CORE.md`](docs/PRODUCT-CORE.md) §7。

**关键提醒:多跳推理必须产品化、结构化(核心对象 `ReasoningChain`,见 PRODUCT-CORE.md §9),不能只写成 AI 长文** —— 否则别人很快抄走;难抄的是"事件库+产业链图谱+推理链对象+每跳references+置信度+资产映射+历史复盘+自选关联"这套系统。

**各角色落地:** 技术—数据/检索/`chainEdges` 都为多跳推理服务,推理产出结构化 ReasoningChain;产品—每屏给"一句话判断 + 对我自选票的影响",信息架构围绕 事件→传导→个股;美术—因果链与产业链热力一眼可读(方向/强弱/冷热)。**⚠️ 反面:纯新闻 feed / 行情看板 / 资讯聚合器。**

## 其它约定
- 产品定位:A 股散户的 AI「盯盘搭子」(陪盯盘的搭子,不是荐股/喊单工具)。
- 合规口径**保守**:不喊单/不荐股/不预测涨跌;战绩讲"联动有效率"复盘、不喊准确率;"受益"→"关联";AI 输出挂"不构成投资建议";关系标"研究框架梳理·非确认"。
- 外部服务/环境变量:见 [`docs/EXTERNAL_SERVICES.md`](docs/EXTERNAL_SERVICES.md)(新增服务先查、接入后更新)。
- 部署:push `main` 自动部署(Vercel,函数区锁东京 hnd1);改库走 `/api/admin/init-db`。
