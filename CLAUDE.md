# StockTell · 项目内核(必读)

> 本文件是所有贡献者(技术 / 产品 / 美术,含 AI 助手)动手前的必读内核。第一条永远是产品的立身之本。

## 🧭 第一内核:AI 时代的 Bloomberg = 事件 → 产业链 → 个股(多跳推理)

**StockTell 要做的是「AI 时代的 Bloomberg」—— 不是股票软件,是「产业理解工具」。**
核心竞争力 = 把「事件 → 产业链 → 个股影响」串成一条**可核实、可推演**的因果链。**第一条:绝不做新闻聚合。**

别人给"发生了什么"(新闻/行情堆叠);我们做**推理**:一个事件如何**逐跳**沿产业链传导,最终落到具体受益个股。

**这才是"推理"(范式例子,牢记):**
```
OpenAI 发布 GPT-6
   ↓ 推理成本下降
   ↓ 算力 / GPU 需求 ↑
   ↓ HBM(高带宽显存)
   ↓ 液冷(散热)
   ↓ 光模块(互联)
   ↓ 国内受益:海光 / 长电 / 中际旭创 …
```
从一个全球事件,**多跳**推演到国内受益标的 —— 这是护城河,新闻聚合抄不走。

**交付三要素(每个场景尽量都给):**
1. **一句话判断** —— 结论先行(这事利好/利空谁、该盯什么),不是丢一堆数据让用户自己想。
2. **产业链热力** —— 一眼看到链条上哪些环节在升温 / 降温。
3. **可核实因果链** —— 每一跳有依据(来源 / 关系边 / 复盘),不编造,守保守合规。

**三条硬规矩(做任何功能/交互/视觉前自问):** ①它强化这条链吗?只"多一条新闻/行情"就不做。②落点永远是"对我(用户自选)的票意味着什么"。③可核实、不编造。

**各角色落地:**
- **技术**:数据/检索都服务这条链——`chainEdges`(产业链边:方向+强弱+依据,是多跳推理的骨架)、「为什么动」(事件→真实来源)、联动雷达(跨市场传导)、个股页(事件+产业链位置+对我的影响)。新数据/接口先问"是否喂养这条链/这次推理"。
- **产品**:每屏给"一句话判断 + 这事对我的票的影响";信息架构围绕 事件→传导→个股,不是新闻时间流。
- **美术**:因果链与**产业链热力**一眼可读(方向/强弱/传导路径/冷热),不是新闻列表堆叠。

**⚠️ 反面(别做成):** 纯新闻 feed、纯行情看板、资讯聚合器、"什么都报但不解释对我的票的影响"。

## 其它约定
- 产品定位:A 股散户的 AI「盯盘搭子」(陪盯盘的搭子,不是荐股/喊单工具)。
- 合规口径**保守**:不喊单/不荐股/不预测涨跌;战绩讲"联动有效率"复盘、不喊准确率;"受益"→"关联";AI 输出挂"不构成投资建议";关系标"研究框架梳理·非确认"。
- 外部服务/环境变量:见 [`docs/EXTERNAL_SERVICES.md`](docs/EXTERNAL_SERVICES.md)(新增服务先查、接入后更新)。
